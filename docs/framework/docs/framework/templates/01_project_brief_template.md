# 01 Project Brief

> **TL;DR:** Template for a one-page executive summary covering what is being built, for whom, and why.
> **Covers:** product summary, primary user, core problem, first value event, v1 scope, tech summary | **Phase:** 2

## Instructions

This is a one-page executive summary of the product. It should be readable in under 2 minutes and give anyone full context on what is being built and why.

## Product Summary

> Example: InvoiceFlow is a web application that helps freelancers create, send, and track invoices from a single dashboard. It replaces the spreadsheet-plus-email workflow with a purpose-built tool that connects directly to Stripe for online payments.

## Primary User

> Example: Independent freelancers and solo consultants managing their own client billing. They have 5-50 active clients and send 10-30 invoices per month.

## Core Problem

> Example: Freelancers waste hours each week switching between spreadsheets, email, and payment platforms to manage invoicing. This leads to late payments, missed follow-ups, and poor cash flow visibility.

## First Value Event

> Example: User successfully creates and sends their first invoice to a real client, receiving a confirmation that the email was delivered.

## V1 Summary

> Example: V1 delivers the core invoice lifecycle — create, send, track, get paid. Includes client management, Stripe payment links, email delivery, a dashboard with payment status, and basic revenue analytics. All pages are mobile responsive.

## Non Goals

> Example: Multi-currency, estimates/proposals, time tracking, team collaboration, white-label branding, and public API are explicitly out of scope for v1.

## Technical Summary

> Example: Next.js (app router) with TypeScript and Tailwind CSS. PostgreSQL with Prisma. Stripe for payments. SendGrid for transactional email. Deployed on Vercel. Auth via email/password with magic link option.
